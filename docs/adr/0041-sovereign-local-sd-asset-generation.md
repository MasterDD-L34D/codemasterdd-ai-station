# ADR-0041 -- Sovereign local-SD asset generation: validated for ornate-decorative, bounded out for flat/functional

- Status: Proposed (Eduardo ratifies)
- Date: 2026-06-18
- Deciders: Eduardo (hub: Claude Code / Opus 4.8, supervised spike)
- Related: ADR-0036 (unified orchestration, local-fleet spoke), ADR-0030 (Hybrid A1 sovereign-first),
  ADR-0023 (post-Max routing). Filone "delegated material generation" (memory project_ferrospora_sd_spike).

## TL;DR

A supervised spike stood up a sovereign local Stable-Diffusion pipeline on the VRAM-rich Ryzen (RTX 4070S)
to test whether the missing Godot-v2 Ferrospora UI assets can be generated by a delegated local channel
instead of the manual ChatGPT-Pro flow. Result: **adopt local-SD as a delegated content-gen spoke for
ORNATE-DECORATIVE single-object assets only** (medallions: 6/8 passed a pre-registered falsification bar).
**Bound it OUT for flat tactical markers and functional UI panels** -- both failed (organic blobs; no
controlled layout) and stay manual. Nothing is promoted; outputs are staging-only behind Eduardo's
per-asset approval.

## Context

- The Godot-v2 Ferrospora UI Art Pass v1 pipeline is prep-complete (queues, prompt templates, validation
  tooling, runtime-drop imported) but 7 Tier1 PNGs are unrendered. The repo's own decision guide
  (`FERROSPORA_IMAGE_PIPELINE_DECISION_GUIDE.md`) routes image generation to **ChatGPT Pro (manual)** and
  **excludes paid image APIs**, while explicitly allowing **local optional** generators (ComfyUI/A1111).
- The filone goal is to generate missing material via a delegated channel (Opus supervises; work runs on
  swarm/Jules/agents/local-fleet). Local-SD is the only channel that produces images without a paid API
  and without the manual ChatGPT-Pro step -- so the open question was empirical: does it meet the bar?
- Standards verified first: SDXL (OpenRAIL++-M) + LayerDiffuse (OpenRAIL-M) permit **commercial use**
  (a second-hand "non-commercial" claim was wrong vs the HF model card); IP-Adapter is Apache-2.0. The one
  hard external obligation is **Steam AI-content disclosure at ship-time**; C2PA Content Credentials are
  recommended provenance. Community signal (gamedev) is strongly anti-"AI slop" + pro-disclosure, which the
  human-review gate + disclosure mitigate.

## Decision

Adopt a **pinned, sovereign local-SD pipeline on Ryzen** as a delegated content-gen spoke, scoped to
**ornate-decorative single-object UI assets**:

- Stack (all pinned): ComfyUI portable **v0.25.0** + SDXL base 1.0 + **LayerDiffuse Conv-Injection**
  (transparent RGBA) + **IP-Adapter style-transfer** (preset VIT-G, ref = an existing canon Ferrospora
  asset). Custom nodes pinned: `huchenlei/ComfyUI-layerdiffuse@b4f6a9e`, `cubiq/ComfyUI_IPAdapter_plus@a0f451a`.
- Install kept at `C:\AI\ferrospora-spike` on Ryzen (reusable; the ComfyUI server is stopped when idle).
- Governance: outputs land in staging (`artifacts/ferrospora_image_intake/`), **never auto-promote**;
  validate -> contact-sheet -> **Eduardo per-asset approval** -> promote -> asset-only PR (merge = Eduardo).
  A different-model vision judge (fleet-tools `cross_check`) is a pre-ship gate (deferred this spike).
  At ship: Steam AI disclosure + optional C2PA provenance sidecar (seed + workflow + model pins).

**Out of scope (stays manual ChatGPT-Pro / designer):** flat tactical board markers and functional UI
panels (UnitInfoPanel / ForecastPanel / overlays). This is the delegability boundary, not a TODO.

## Evidence (pre-registered falsification, SDMG)

Pre-registered bar: per asset, generate N=8 seeds; each scored on 4 binary hard-gates (true-alpha
transparent / no-text / centered+complete / single-isolated) + style-fidelity 0-1 vs the canon reference.
Pass = >=4/8 meet all hard-gates AND fidelity >=0.6.

- **ct_medallion: PASS.** Transparency 8/8 (alpha_min 0, 30-76% transparent margin); single+circular+no-text
  ~6/8; style-fidelity high (ornate bronze/oxidized-metal/teal-gem, on-brand). Best seeds: 00005, 00007.
- **Scale to 6 Tier1: FAIL/marginal.** board_overlays (4 assets x 3 seeds) = **0/12** usable -- rendered as
  organic blobs/foliage, color-coding failed (the "fungal biotech" prompt + ornate style-ref produced mush,
  not flat glyphs). combat_panels (2 x 3) = **1-2/6** marginal (one plausible vertical frame; the rest drift
  to rings/medallions/a landscape).

Interpretation: the pipeline reproduces an ornate decorative single object well (it matches the ornate
style-ref) but cannot produce (a) flat geometric iconography or (b) controlled functional UI layouts -- both
are different generation problems requiring different tooling (flat-icon model/no-ornate-ref; or a designer).

## Technical findings (durable, save a future session the grind)

1. **Attention-Injection conflicts with IP-Adapter.** Both patch attention; LayerDiffuse "SDXL, Attention
   Injection" + IP-Adapter yields opaque output (no alpha cutout). Fix: **"SDXL, Conv Injection"** (needs a
   separate `layer_xl_transparent_conv.safetensors`, ~3.37GB, auto-downloaded on first use).
2. **Pin can be incompatible with ComfyUI core.** The pinned layerdiffuse calls core `JoinImageWithAlpha`'s
   old instance method, removed by ComfyUI v0.25.0's node-v3 migration. Fix: **rewire the workflow**
   (`LayeredDiffusionDecode` -> `InvertMask` -> core `JoinImageWithAlpha`) rather than patch third-party code.
3. **Verify node schemas via `/object_info`** before authoring a workflow; IP-Adapter preset must match the
   downloaded models (VIT-G for `ip-adapter_sdxl` + CLIP-ViT-bigG; "STANDARD" expects absent ViT-H files).

## Consequences

- A sovereign, paid-API-free art-gen capability exists for ornate-decorative assets, reusable for future
  Ferrospora ornate work; the local-fleet spoke (ADR-0036) is extended with image generation (bounded).
- The validated medallions are delivered to staging (not promoted); promotion is a separate Eduardo-gated
  PR. Overlays + functional panels remain on the manual track.
- Supply-chain: running pinned third-party SD code on Ryzen was gated by the auto-mode classifier and
  authorized explicitly by Eduardo (lock-commit-at-import). The pins are recorded above.
- Negative result is a result: this ADR records the boundary so the filone does not re-attempt local-SD for
  flat/functional assets.

## Alternatives considered

- **Manual ChatGPT-Pro only** (repo default): higher quality, but not delegable / not automatable; remains
  the path for flat/functional assets.
- **Attention-Injection (rejected):** broken transparency under IP-Adapter (see findings).
- **Patch the pinned third-party node (rejected):** workflow rewire avoids mutating pinned code.
- **rembg/BiRefNet post-process for alpha (deferred fallback):** unnecessary once Conv-Injection produced
  clean alpha; kept as a fallback for cases where LayerDiffuse underperforms.
