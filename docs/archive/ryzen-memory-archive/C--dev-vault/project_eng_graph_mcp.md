## Fix 2026-05-26 — MCP transport (commit `b4c46b6b5`)

After restart the server loaded but `eng_graph_query` FAILED: `Subprocess init timed out after 60.0s` (and earlier `[Tool result missing due to internal error]`). 3 stacked causes, fixed at the root:

1. **stdout pollution** — cognee/transformers/HF print to stdout, corrupting the stdio JSON-RPC stream. `mcp_server.py` now wraps cognee work in `contextlib.redirect_stdout(sys.stderr)`.
2. **orphan-lock** — cognee 1.1.0 `db_workers` are `multiprocessing` daemons. Crashed/killed clients (e.g. a flaky background `query.py`) leave **orphaned workers holding an exclusive lock** on `cognee_graph_ladybug` → every later DB open fails with "Could not set lock on file". Cleanup recipe: `Get-CimInstance Win32_Process -Filter "Name='python.exe'"` and kill any with cmdline `multiprocessing.spawn ... spawn_main(parent_pid=...)` whose parent is dead.
3. **spawn-deadlock (core)** — `graph_database_subprocess_enabled` defaults to **True** → kuzu/lancedb run in spawned worker processes. On **Windows**, spawning a worker from inside FastMCP's already-running `ProactorEventLoop` deadlocks the spawn bootstrap handshake → worker never signals READY → 60s init timeout. Standalone `query.py` worked because it spawns from a clean main-thread `asyncio.run()`.

**Fix (root):** `_sovereign.configure()` now sets `set_graph_database_subprocess_enabled(False)` + `set_vector_db_subprocess_enabled(False)` → kuzu + lancedb run **in-process**. No spawn → no deadlock → no orphan-lock daemon. Safe here (small read-only graph, infrequent queries → negligible native-memory growth — the only reason subprocess mode exists). Verified: smoke `isError=False` ×3 + 2 live Claude MCP calls OK; **0 orphan workers** after server death (in-process child reaped with parent). Startup ~6s; query ~90s (local 14b GRAPH_COMPLETION floor).

**Transport smoke:** `Vault-ops-remote/scripts/eng_cognee/_smoke_mcp.py` spawns the server via the mcp stdio client and does a real handshake + tool call (captures server stderr to `%TEMP%/engq_server_stderr.txt`). Use it to validate transport WITHOUT restarting Claude. Pre-warm Ollama (`qwen2.5-coder:14b` + `nomic-embed-text`) with `keep_alive` to avoid the ~120s cold model-load.

**Anti-pattern (candidate global #20):** cognee/db_workers subprocess-daemon + active asyncio loop + Windows = spawn-deadlock + orphan-lock. For long-running async services on Windows with a small graph, prefer in-process DB adapters.
