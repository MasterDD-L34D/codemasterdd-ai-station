# SPRINT_00_BOOTSTRAP

Questo sprint serve a rendere il progetto davvero operabile da Claude Code.

## Obiettivo sprint

Creare o normalizzare il minimo sistema operativo del progetto così che le sessioni future possano essere più autonome, più coerenti e meno dispersive.

## Deliverable obbligatori

- file bootstrap esistenti o normalizzati
- file del pacchetto operativo Claude Code presenti
- indice dell'archivio leggibile
- repo map iniziale
- compact context aggiornato
- backlog prioritizzato
- open decisions iniziale

## Task

### Task 1 — Audit file di progetto
- verificare file bootstrap esistenti
- trovare duplicati o versioni stale
- segnare gap reali

### Task 2 — Normalizzazione bootstrap
- creare o aggiornare:
  - PROJECT_BRIEF.md
  - COMPACT_CONTEXT.md
  - DECISIONS_LOG.md
  - BACKLOG.md
  - MODEL_ROUTING.md
  - FIRST_PRINCIPLES_GAME_CHECKLIST.md
  - OPEN_DECISIONS.md

### Task 3 — Audit pacchetto operativo
- verificare che i 4 file operativi siano presenti e coerenti
- adattarli al repo reale se necessario

### Task 4 — Repo map iniziale
- creare mappa cartelle
- individuare core vs supporto vs cerimonia
- segnare zone ad alto rischio

### Task 5 — Primo backlog vero
- definire priorità alta/media/bassa
- definire il primo sprint tecnico reale

## Definition of Done

Lo sprint è completo quando:
- Claude Code può leggere il progetto senza dipendere da chat pregresse
- i file canonici sono presenti e usabili
- esiste almeno una repo map iniziale
- esiste almeno un primo sprint tecnico reale

## Note

Questo sprint non serve a “migliorare il gioco” direttamente.
Serve a costruire il binario su cui il lavoro successivo potrà correre bene.
